# AI 胶卷推荐说明

StillLight 当前的胶卷推荐是本地启发式能力。它不会上传图片，不调用云端服务，不使用大模型，也没有引入新的第三方依赖。推荐逻辑集中在 `StillLight/AI/FilmRecommender.swift`，用于暗房导入图片后给出胶卷候选和解释。

## 当前能力

导入图片后，推荐器会把图片缩小到 48x48 像素，在设备本地采样 RGB 像素，提取一组轻量指标，再根据固定规则为胶卷预设打分。

当前提供两类 API：

- `recommend(image:presets:)`：保留原有单一推荐 API，返回最优胶卷、推荐理由和图片指标。
- `recommendations(image:presets:limit:)`：返回 Top N 候选，默认 Top 3。每个候选包含胶卷 preset、分数、短理由、中文短理由、原始指标和用于展示的指标摘要。

旧 API 会复用 Top 3 的第一项，因此现有调用方不需要修改。

## 输入指标

`ImageMetricsAnalyzer` 目前只读取图像自身的基础视觉特征：

- 亮度 `brightness`：基于 RGB 线性加权亮度的平均值。
- 饱和度 `saturation`：逐像素计算最大通道和最小通道差值，再取平均。
- 冷暖 `warmth`：红色通道减蓝色通道的平均值，数值越高越偏暖，越低越偏冷。
- 反差 `contrast`：每个像素亮度与平均亮度的平均偏差。

这些指标会被整理为展示摘要，例如 `B 52 / S 28 / W 3 / C 14`，方便 UI 或调试信息解释推荐依据。

## 打分方式

推荐器内置一组 `FilmRecommendationRule`。每条规则对应一个胶卷 ID、英文短理由、中文短理由和一个本地评分函数。评分函数只依赖 `ImageMetrics`，不会读取额外上下文。

当前规则覆盖的主要场景：

- 低光且高反差：优先 `human-vignette-800`，强调边缘压暗和中心聚焦。
- 低光：优先 `tungsten-800`，保留暖色高光和轻微泛光。
- 低饱和且高反差：优先 `silver-hp5`，转向黑白纪实。
- 明亮且低饱和：优先 `muse-portrait-400`，保护高光和柔化肤色。
- 偏冷：优先 `green-street-400`，强化青绿色街拍氛围。
- 高饱和且高反差：优先 `pocket-flash`，靠近一次性闪光机感。
- 明亮且偏暖：优先 `sunlit-gold-200`，走暖色日光彩负。
- 均衡光色：优先 `human-warm-400`，作为稳定的人文暖调默认候选。

评分由两类小函数组成：

- `highScore`：某个指标越超过阈值，得分越高。
- `closenessScore`：某个指标越接近目标值，得分越高。

每条规则都有基础分和指标权重。Top 3 会按分数从高到低排序。如果当前传入的胶卷库缺少某些规则里的 preset，推荐器会跳过这些规则；如果候选不足，会用胶卷库里的其他 preset 作为中性 fallback 补足列表。

## 为什么仍是启发式

MVP 阶段优先保证：

- 离线可用，导入暗房时不依赖网络。
- 延迟低，批量导入时逐张快速完成。
- 可解释，产品文案能直接对应亮度、饱和度、冷暖、反差。
- 风险小，不引入模型包体、权限、隐私和推理性能问题。

这套逻辑不能识别人像、天空、食物、夜景、建筑等语义内容，也不会理解构图主体。它只是根据低层视觉指标选择“更可能合适”的胶卷风格。

## 后续路线

后续可以把推荐能力分三步升级：

1. Vision 场景辅助：使用 Apple Vision 在本地识别人脸、主体、地平线、文字、矩形建筑等信号，把语义标签加入评分，但仍保留当前指标作为基础。
2. CoreML 本地模型：训练轻量分类或排序模型，输入图像缩略图、基础指标和 Vision 标签，输出候选胶卷分数。模型只在设备本地运行。
3. 解释层稳定化：无论底层是规则还是模型，都保留可解释字段，例如命中的关键指标、场景标签、分数拆分和推荐短理由，避免 UI 只能展示黑盒结论。

短期内推荐器应继续避免云端推理和大模型依赖，直到隐私、延迟、成本和离线体验都有明确产品收益。
