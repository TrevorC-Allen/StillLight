# StillLight 中文演示脚本

本文档用于 Notion 作品集、面试现场和 3 分钟录屏。目标是把 StillLight 讲成一个真实可跑的 iOS 胶片相机闭环：拍照、暗房、本地推荐、保存和相册记录。

## 演示顺序

```text
拍照
-> 暗房
-> 本地 AI 推荐
-> 原生视频
-> 相册 / 系统照片
```

这条顺序优先展示当前已经完成的照片主闭环。胶片视频效果、云端 AI 和 Metal 实时预览只作为后续路线或技术 spike 讲，不说成已完成能力。

## 3 分钟演示路径

| 时间 | 屏幕操作 | 讲解重点 |
| --- | --- | --- |
| 0:00 - 0:20 | 打开 StillLight，确认中文 UI，停留在相机页 | “StillLight 是一个原生 iOS 胶片相机。它不是只做滤镜预览，而是把拍摄、冲洗、保存和本地相册记录做成一个完整闭环。” |
| 0:20 - 0:45 | 打开胶卷选择器，切到主推胶卷，选择 Human Warm 400 | “胶卷不是图片素材，而是一组可解释参数：曝光、色温、Tone Curve、Halation、暗角、颗粒、边框和日期戳。现在内置 27 个原创胶片 / 机型风格。” |
| 0:45 - 1:15 | 回到取景页，展示网格、水平仪、比例、0.5x / 1x / 3x 或捏合缩放，点按对焦，调曝光 | “相机层走 AVFoundation：实时预览、拍照、焦点 / 测光、曝光补偿、多摄变焦和构图辅助都在原生相机会话里完成。” |

| 1:35 - 2:10 | 进入暗房，导入多张已有照片，展示横向队列，选择冲洗当前 / 冲洗全部 | “暗房复用同一套胶片管线，支持多图导入和批量冲洗。这里体现的是从相机 App 到个人暗房的工作流，不只是单张滤镜处理。” |

| 2:45 - 3:00 | 打开 App 内相册 / 系统照片，展示最新照片、边框、日期戳和胶卷标注 | “最后成片会写入本地记录，并在授权时保存到系统照片。作品集重点是：原生相机、可解释图像管线、暗房批处理和清楚的后续技术路线。” |

## 逐步屏幕操作

### 1. 拍照

1. 打开 StillLight。
2. 如果不是中文，进入设置切换中文。
3. 回到相机页，允许相机、麦克风和照片权限。
4. 打开胶卷选择器，选择 `Human Warm 400`。
5. 展示构图辅助：网格线、水平仪、比例取景框。
6. 展示相机交互：点按对焦、曝光补偿、捏合缩放或镜头按钮。
7. 按下快门。
8. 打开最近照片 / 冲洗结果。
9. 保存到系统照片。

可讲技术点：

- `AVCaptureSession` 负责真机预览和拍摄。
- 对焦、测光、曝光补偿和变焦都属于原生相机控制层。
- 胶片效果在拍照后处理，不承诺实时胶片预览。
- 照片管线使用 CoreImage 加 CPU 颗粒，优先保证输出稳定。

### 2. 暗房

1. 进入暗房。
2. 从 PhotosPicker 选择 3 到 5 张照片。
3. 展示横向队列和当前选中照片。
4. 点击冲洗当前。
5. 再点击冲洗全部。
6. 保存当前或保存全部。

可讲技术点：

- 暗房和拍照直出复用同一套 `FilmImagePipeline`。
- 多图处理按队列逐张执行，避免一次性并发处理大图带来的内存压力。
- 暗房是作品集里最能说明产品闭环的部分：已有照片也可以统一冲洗。

### 3. 本地 AI 推荐

1. 在暗房导入照片后停留在推荐区域。
2. 展示推荐胶卷和推荐理由。
3. 手动切换另一个胶卷，说明用户仍然有控制权。

可讲技术点：

- 当前推荐是本地启发式，不是云端大模型。
- 输入特征来自缩略采样后的亮度、饱和度、冷暖和反差。
- 推荐失败或不合适时，用户仍然可以手选胶卷。
- 后续路线是 Vision / CoreML 或更完整的候选排序，不写成当前已完成。

### 4. 原生视频

1. 回到相机页。
2. 切换到视频模式。
3. 录制 2 到 3 秒。
4. 停止录制，确认保存。

可讲技术点：

- 当前视频是原生 `AVCaptureMovieFileOutput` 录像闭环。
- 已有能力是录制、计时、麦克风和保存到系统照片。
- 胶片视频效果不是当前完成能力；后续需要 Metal 实时处理或 AVAssetWriter 离线导出。

### 5. 相册

1. 打开 App 内相册或最近照片入口。
2. 展示刚保存的成片。
3. 再打开系统照片，展示导出结果。
4. 放大看边框、日期戳和胶卷标注。

可讲技术点：

- App 内维护本地照片记录，用于作品回看、胶卷信息和收藏状态。
- 系统照片保存使用 Photos 权限，适合真实用户工作流。
- 输出不是简单套滤镜，而是把胶卷参数和边框仪式感写进成片。

## 失败时备用说法

| 场景 | 现场说法 | 接下来操作 |
| --- | --- | --- |
| 相机权限没开 | “这里是 iOS 权限路径，真实相机 App 必须显式申请相机、麦克风和照片访问。权限未开时，我会先走设置授权。” | 打开设置授权，或切到已录好的拍照片段 |
| 设备锁屏 / Xcode 启动失败 | “这是 iPhone 真机调试常见问题，不是业务链路失败。设备需要保持解锁并信任开发者证书。” | 解锁设备，重新启动 App |
| 拍照保存失败 | “照片处理和系统相册写入是两个阶段。即使系统照片权限失败，也可以说明本地处理链路和权限错误提示。” | 展示 App 内结果或本地相册记录 |
| 暗房导入失败 | “暗房依赖 PhotosPicker 和照片权限。如果导入失败，我会直接展示预先导入的一组样张，重点讲同一管线如何批量冲洗。” | 切到已有样张或录屏 |
| 推荐结果不理想 | “推荐是辅助，不替代用户选择。当前版本是本地启发式，所以我更关注它可解释、可关闭、可被用户覆盖。” | 手动切换主推胶卷 |
| 视频胶片效果被追问 | “当前完成的是原生录像闭环。胶片视频需要逐帧处理、实时预览和编码导出，后续会走 Metal / AVAssetWriter spike，不在 MVP 里夸大。” | 展示原生录像并回到照片管线 |
| Metal 实时预览被追问 | “当前照片管线先用 CoreImage 保证稳定导出。Metal 实时预览是后续性能路线，适合在管线稳定后迁移。” | 回到成片和暗房输出 |
| 云端 AI 被追问 | “当前没有云端 AI。推荐完全在本地完成，后续才考虑 Vision / CoreML 或云端多模态方案。” | 展示本地推荐理由 |

## 结尾话术

“StillLight 当前最完整的价值是照片主闭环：原生相机拍摄、可解释胶片管线、暗房批量冲洗、本地推荐和相册保存。它把一个看起来像滤镜 App 的东西，拆成了相机控制、图像处理、Photos 工作流、本地数据和后续 GPU / AI 路线。未完成的部分我会明确标成路线：胶片视频、Metal 实时预览和云端 AI 都不会在作品集里说成已经完成。”
