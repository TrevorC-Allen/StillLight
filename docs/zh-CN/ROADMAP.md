# StillLight 4 周开发路线

本文档用于 Notion 作品集、面试复盘和后续开发排期。它只描述当前 StillLight 已有闭环之上的下一步，不把未完成能力写成已完成能力。

## 当前基线

| 模块 | 已有能力 | 可继续强化的点 |
| --- | --- | --- |
| 相机 | AVFoundation 实时预览、拍照、前后摄切换、闪光灯、曝光补偿、点按对焦、捏合缩放、多摄镜头按钮、网格线、水平仪、比例取景框 | 拍摄状态更稳、权限和错误提示更完整、真机边界场景记录更充分 |
| 胶片管线 | CoreImage 处理曝光、色温、色调、对比度、饱和度、Tone Curve、Halation、暗角、漏光；CPU 颗粒；日期戳、白框、相纸和拍立得边框 | 参数回归样张、强度控制、3D LUT / Metal 实时预览路线 |
| 暗房 | PhotosPicker 多图导入、当前 / 全部冲洗、当前 / 全部保存、横向队列、共享胶片管线 | 批量取消、细分进度、失败项重试、原图 / 成片对比 |
| AI | 本地启发式推荐，根据亮度、饱和度、冷暖和反差推荐胶卷 | 推荐理由可视化、候选胶卷排序、Vision / CoreML 版本预研 |
| 视频 | 原生 `AVCaptureMovieFileOutput` 录像、麦克风、录制计时、保存到系统照片 | 胶片视频预览和导出，后续走 Metal / AVAssetWriter |
| 性能 | 输入降采样到 2400px、异步导出、暗房逐张处理、本地 JSON 记录 | 批量内存曲线、低端机降级策略、管线耗时埋点 |
| 作品集包装 | 中文 / 英文 README、MVP、架构、预设、真机运行手册 | 路线图、演示脚本、对比样张、面试亮点证据表 |

## 优先级定义

| 优先级 | 含义 | 判断标准 |
| --- | --- | --- |
| P0 | 作品集演示必须稳定 | 真机拍照、冲洗、保存、暗房批量和文档证据不能断 |
| P1 | 面试技术深度加分 | 能讲清 AVFoundation、CoreImage、Photos、批处理、AI 推荐或性能取舍 |
| P2 | 后续探索和包装增强 | 有价值但不影响当前演示闭环，适合放在路线图后段 |

## 先后顺序

| 顺序 | 优先级 | 先做什么 | 后做什么 | 原因 |
| --- | --- | --- | --- | --- |
| 1 | P0 | 相机拍照闭环、导出、相册记录、暗房批量稳定性 | 新增大功能 | 当前作品集价值来自“真实可跑、可拍、可保存” |
| 2 | P0 | 胶片管线回归样张、主推胶卷视觉稳定 | 更多胶卷数量 | 27 个预设已经足够，接下来要保证每个主推场景能展示差异 |
| 3 | P1 | 暗房进度、取消、失败恢复、原图对比 | 云同步或账号系统 | 暗房最能体现从相机 App 到个人暗房的产品完整度 |
| 4 | P1 | AI 推荐解释、性能测量、Metal 原型验证 | 完整 CoreML 训练或云端多模态 | 先把当前启发式讲清楚，再升级模型路线 |
| 5 | P2 | 胶片视频技术验证、作品集包装、演示材料 | 商业化功能 | 视频是技术亮点，但不能压过照片主闭环 |

## 模块路线

### 相机

| 优先级 | 方向 | 具体工作 | 验收 |
| --- | --- | --- | --- |
| P0 | 稳定拍摄 | 回归权限、会话启动、拍照、焦点、曝光、缩放、镜头按钮 | 真机连续拍摄不破坏 UI 状态 |
| P1 | 交互细节 | 改善异常提示、拍照状态、最近照片入口 | 用户知道当前能否继续拍 |
| P2 | 拍摄辅助 | 更多构图辅助或可配置网格 | 不影响核心拍摄体验 |

### 胶片管线

| 优先级 | 方向 | 具体工作 | 验收 |
| --- | --- | --- | --- |
| P0 | 输出一致性 | 主推胶卷样张回归，避免参数改动导致风格漂移 | 三个主推胶卷有稳定对比证据 |
| P1 | 可解释参数 | 把曝光、色温、Tone Curve、Halation、暗角、颗粒映射到视觉说明 | 面试能按管线顺序讲清楚 |
| P2 | GPU 升级 | 预研 3D LUT、Metal 颗粒、Metal 暗角 | 有方案或最小原型，不影响现有导出 |

### 暗房

| 优先级 | 方向 | 具体工作 | 验收 |
| --- | --- | --- | --- |
| P0 | 批量可用 | 多图导入、冲洗全部、保存全部稳定 | 多张图处理结果和本地记录数量一致 |
| P1 | 长任务体验 | 进度、取消、重试、失败原因 | 长批处理可控，不像卡死 |
| P2 | 编辑能力 | 每张图独立强度、比例和推荐记录 | 让暗房从批处理走向轻编辑 |

### AI

| 优先级 | 方向 | 具体工作 | 验收 |
| --- | --- | --- | --- |
| P0 | 保持本地推荐稳定 | 不让推荐失败影响暗房导入 | 推荐失败时仍可手选胶卷 |
| P1 | 可解释推荐 | 显示亮度、饱和度、冷暖、反差和推荐理由 | 面试能讲清启发式和下一步模型化 |
| P2 | Vision / CoreML | 引入场景标签、人像检测、夜景识别 | 作为后续升级，不影响当前 MVP |

### 视频

| 优先级 | 方向 | 具体工作 | 验收 |
| --- | --- | --- | --- |
| P0 | 原生录像稳定 | 保持麦克风、计时、临时文件、Photos 保存稳定 | 能录、能存、能回看 |
| P1 | 技术方案 | 拆分实时预览、离线渲染、编码导出方案 | 能解释为什么不能简单复用照片管线 |
| P2 | 胶片视频原型 | Metal + `AVAssetWriter` 最小链路 | 原型可展示，但不承诺进入主线 |

### 性能

| 优先级 | 方向 | 具体工作 | 验收 |
| --- | --- | --- | --- |
| P0 | 避免明显卡顿 | 保持降采样、异步导出、逐张批处理 | UI 不被单张导出长期阻塞 |
| P1 | 数据化 | 记录单张处理耗时、批量耗时、内存峰值 | 有表格可放进 Notion |
| P2 | 低端机策略 | 降低最大边长、颗粒强度或批处理并发 | 老设备有明确降级路线 |

## 演示路径

```text
打开 StillLight
-> 切换到中文 UI
-> 选择 Human Warm 400
-> 展示网格、水平仪、比例和变焦按钮
-> 拍一张照片
-> 查看最近照片和本地 Gallery 记录
-> 进入暗房导入多张照片
-> 展示本地 AI 推荐和推荐理由
-> 批量冲洗并保存
-> 展示系统照片、边框、日期戳和胶卷标注
-> 说明照片管线为何先用 CoreImage，视频胶片为何放到 Metal / AVAssetWriter 后续路线
```

## 风险和处理

| 风险 | 优先级 | 处理方式 | 不做什么 |
| --- | --- | --- | --- |
| 真机设备差异导致相机能力不一致 | P0 | 在文档记录测试机型，多摄按钮按能力显示 | 不硬编码所有设备都有 0.5x / 3x |
| 批量暗房内存过高 | P0 | 继续逐张处理，记录最大尺寸和耗时 | 不一次性并行处理大图 |
| 胶片参数越调越散 | P1 | 用主推胶卷样张做回归 | 不继续盲目增加预设数量 |
| AI 推荐被误解成大模型能力 | P1 | 明确当前是本地启发式，后续才是 Vision / CoreML | 不在作品集里写“云端 AI 已完成” |
| 胶片视频投入过大 | P2 | 先做技术 spike 和方案拆分 | 不在照片 MVP 稳定前重写视频链路 |
